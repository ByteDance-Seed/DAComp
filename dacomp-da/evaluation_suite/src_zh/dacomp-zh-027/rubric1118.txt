## 需求1：持续负面影响的“分级量化分析”（本需求最高可得 13分）
### 1.1 数据口径与等级映射（3 分，AND） 
- **1.1.A 字段映射 + 等级口径 + 复现信息（3 分）** 
 - *1.1.A.1 完备性（1 分）*：给出**原始→分析字段映射** （含上述关键字段），并说明**时间范围与样本量 N** （≥10条抽样核对映射）； 
 **合格阈值**：关键字段均出现；时间范围与 N 明确。 
 - *1.1.A.2 精确性（1 分）*：说明 L1–L5 **互斥** 定义（任一来源或自定义，但边界**不重叠**），能据此判定样本； 
 **合格阈值**：L1–L5 覆盖、互斥、可判定。 
 - *1.1.A.3 复现与清洗日志（1 分）*：记录**缺失/异常/单位** 处理，给出**剔除比例、原因、最终 N**；标注**代码版本/时间/输入 md5**。 
### 1.2 指标计算与异常处理（3 分，OR） 
- **1.2.A 阈值法 + Winsor 化（3 分）** 
 - *1.2.A.1 计算公式（1 分）*： 
 `share_high_disease(Lk) = count(disease_risk='High'|level=k)/N_k` 
 `share_mh_low(Lk) = count(mh_access<10/万人|level=k)/N_k` 
 `share_wq_low(Lk) = count(wqi<50|level=k)/N_k`（标注单位及红阈值）； 
 `mean(co2_te|level=k)` 等。 
 - *1.2.A.2 异常处理（1 分）*：连续变量（`wqi, co2_te, mh_access`）`winsorize 1%/99%`；类别/比例保持原值，缺失**从分母剔除并报告 N_k**；输出 winsor 区间/剔除比例/最终 N_k。 
 - *1.2.A.3 等级×指标表（1 分）*：提供 **L1–L5 ×（share_high_disease, share_wq_low, mh_access均值, recycling率, co2均值, displaced均值）**的完整表（含单位）。 
- **1.2.B 分箱 + Bootstrap CI（3 分）**（**等价满足**1.2.A）： 
 对每等级计算**均值/中位数/95% CI（bootstrap≥1000）**并报告 CI。 
### 1.3 跨等级差异与显著性（4 分，AND） 
- **1.3.A 检验 + 效应量 + 稳健性 + 结论（4 分）** 
 - *1.3.A.1 显著性检验（2 分）*： 
 **任选其一**： 
 - 正态/方差齐：`ANOVA + Tukey`（给出 F、df、p、η²）； 
 - 否则：`Kruskal–Wallis + Dunn–Bonferroni`（给出 H、df、p、ε²）； 
 - **等价**：`置换检验`或**95% CI 不重叠**也可判定显著（需明确效应量如 Cliff’s δ、Hedges g、或η²/ε²）。 
 至少覆盖 2 个核心指标（如 `share_wq_low`、`mh_access`、`displaced均值`、`co2均值`）。 
 - *1.3.A.2 稳健性（1 分）*：报告**中位数排序与均值排序**是否一致；如不一致，给出 ≤80 字的“偏斜/重尾”解释。 
 - *1.3.A.3 结论性（1 分）*：从以下**结论全集**中至少命中 1 条，并**括注证据**（p/CI/均值等具体数值）： 
 - L5 对人最重：`displaced均值(L5)≥L4 且 二者>其它等级 且 p<0.05`； 
 - L4 碳排峰值：`mean(co2_te)(L4)` 最高且 `p<0.05`； 
 - L1–L2 基础环境短板：`share_wq_low(L1或L2)≥30%` 且与 `L3–L4` 有显著差； 
 - 心理缺口在 L5 最大：`share_mh_low(L5)` 最高且 `p<0.05`； 
 - 中度瓶颈（L3）：`供应受阻率(L3)` 最高且 `p<0.05`（如有该指标）。 
### 1.4 可视化与离群/重尾识别（3 分，AND） 
- *1.4.A.1 图表（1 分）*：至少 **3 张**（**功能等价即可**）： 
 ① L1–L5 指标对比（分组柱或雷达）；② `wqi` 箱线或小提琴（按等级）；③ `co2_te` 直方图（标明 bin 宽）。图需**标题/轴单位/图例**齐全。 
- *1.4.A.2 离群识别（1 分）*：按**IQR 规则**逐等级识别连续变量离群，导出**离群清单**（`event_id, level, 指标, 上/下界`）。抽检 3 条给出手算一致性说明（可简述）。 
- *1.4.A.3 分布判别（1 分）*：命中以下任一并给出对应比例/数值： 
 - 右偏/重尾：`mean>median` 且上界离群≥2%； 
 - 左偏：`mean<median` 且下界离群≥2%； 
 - 近似对称：`|mean-median|/std≤0.1` 且双尾离群<1%。

## 需求2：分级建议（SOP）与可执行触发器（本需求最高可得 13分）
### 2.1 风险—措施矩阵 **或** 风险评分卡（4 分，OR） 
- **2.1.A 等级×人群×环境矩阵（4 分）** 
 - *2.1.A.1 矩阵完备（1 分）*：输出 **5×(≥6)** 表（每等级≥**人群3项**：卫生可及、心理触达、迁移/安置；**环境3项**：水质、回收、碳排）。每格含**动作/负责人/时限**。 
 - *2.1.A.2 阈值与触发器（2 分）*：至少写全 **5 个阈值** 并与动作**绑定**；允许直接采用默认阈值（卫≥60/50–60/<50；心理≥15/10–15/<10；WQI≥55/50–55/<50；回收≥50/40–50/<40；单位安置碳排≤120/120–150/>150；受阻率<60/60–70/>70；缺货率<5/5–10/>10）。 
 - *2.1.A.3 优先级规则（1 分）*：提供 `priority = argmax{救命性→收益最大(pop_affected×ΔKPI)→公平性(gini of coverage)}` 的**1 个分配示例**（≤80字具体）。 
- **2.1.B 风险评分卡（4 分，等价满足）** 
 定义 `R = 0.4·z(displaced) + 0.3·z(wqi<50) + 0.2·z(mh_access<10) + 0.1·z(co2_te)`；给出**分数→动作**映射（如 `R≥2` 触发“绿色通道+心理军团化+低碳重建包”）。 

### 2.2 可执行建议与落地清单（4 分，AND） 
- *2.2.A.1 动作=触发器+目标+时限（2 分）*：至少 **6 条**（覆盖 L1–L5 且含“人/环境”两类），每条包含：**触发条件（阈值）→ 动作（数量/频次/地点/负责人） → 目标值（KPI 数字） → 时限**。 
- *2.2.A.2 资源与依赖（2 分）*：为每条动作列出**资源清单**（人员/车辆/设备/预算范围）与**外部依赖**（通行证、跨区协同、数据接口）。缺1要素扣1分。 
### 2.3 可行性与敏感性校验（3 分，AND） 
- *2.3.A.1 约束校验（1 分）*：给出**运力/预算/人手**三类约束的估算与**覆盖率 = 可用资源 / 动作需求**；至少 2 条动作的**计算示例与数值**。 
- *2.3.A.2 敏感性（±20%）（1 分）*：对关键阈值（例：受阻率 70%、心理 10/万人、碳排 150）做 **±20%** 摆动，报告**触发变化**与**目标可达性变化**（表格或要点）。 
- *2.3.A.3 取舍声明（1 分）*：从“**救命性/收益最大/公平性**”三维，给出本批次**优先级排序**与**1条例证**。 
### 2.4 结论全集与证据对齐（2 分，AND） 
- *2.4.A.1 四块内容（1 分）*： 
 ① **等级差异总结**（命中 1.3.A.3 条目，带 p/CI/均值）； 
 ② **人群优先**（卫生/心理/迁移各 1 句，带阈值与目标数）； 
 ③ **环境优先**（水质/回收/碳排各 1 句，带阈值与目标数）； 
 ④ **供应与资源**（受阻/缺货各 1 句，带阈值与目标数）。 
- *2.4.A.2 证据对齐（1 分）*：每条结论**括注证据**（如：“L3 受阻率最高 72.9%，K-W p=0.01；目标<60%（4周）”）。缺数字不得分。

## 需求3：监测看板与复用交付（SOP可视化与数据管道）（本需求最高可得 4分）
### 标准3.1【最高2分】：指挥看板原型与字段定义（OR）
#### 路径3.1.A【2分 | 静态原型 + 字段字典】
- 小标准3.1.A.1【1分｜完备性】模块与布局
  - 提交“一页式看板”原型（低保真也可），至少包含 4 个模块：
    - ① KPI 总览（卫生可及、心理触达、水质<50占比、回收率、单位碳排、供应受阻率、缺货率、安置人口）；
    - ② 热力图（供应断点/水质低值点/心理高风险社区）；
    - ③ 触发器告警区（红/黄/绿阈值与最近 7 日轨迹）；
    - ④ 行动闭环（问题→责任单位→措施→截止→状态）。
  - 合格阈值：四个模块齐全且各自有数据位占位。
- 小标准3.1.A.2【1分｜精确性】字段与计算口径
  - 必须提供字段字典（字段名/类型/单位/口径/刷新频率/来源）：
    - 例：`health_coverage`(float, %, 10天内获得基本诊疗人口÷受灾人口×100, 逐日, 卫健局报表)；
    - `mh_reach_per_10k`(float, 人/万人, 心理服务触达人数÷人口×10000, 周更, 心理专班)；
    - `wqi_low_share`(float, %, WQI<50 点位÷监测点×100, 周更, 水务)；
    - `recycling_rate`(float, %, 回收量÷总固废量×100, 日更, 城管)；
    - `co2_per_household`(float, kgCO₂e/户·月, 重建期排放÷安置户数, 月更, 承包商/电力单位)；
    - `supply_block_rate`(float, %, 关键节点失效÷节点总数×100, 日更, 交通/仓配)。
  - 合格阈值：至少6 个字段给出完整口径与刷新频率。

#### 路径3.1.B【2分 | 可执行的 SQL/伪SQL 指标生成】
- 示例（可直接采用）：
  ```sql
  -- 水质<50占比（周）
  SELECT level, week,
  SUM(CASE WHEN wqi < 50 THEN 1 ELSE 0 END)*1.0/COUNT(*) AS wqi_low_share
  FROM water_quality
  WHERE sample_time BETWEEN :week_start AND :week_end
  GROUP BY level, week;

  -- 心理触达（每万人）
  SELECT region,
  SUM(psych_service_people)*10000.0 / MAX(population) AS mh_reach_per_10k
  FROM mh_services
  WHERE date BETWEEN :d1 AND :d2
  GROUP BY region;
  ```
- 合格阈值：至少给出2 段可运行或可直接改造的 SQL/伪SQL，并注明输入时间窗与分组键。

### 标准3.2【最高2分】：数据管道与触发自动化（OR）
#### 路径3.2.A【2分 | 触发器与任务流（伪算法）】
- 小标准3.2.A.1【1分｜精确性】触发器定义
  - 必须绑定阈值→动作，示例如下（可直接使用）：
    ```
    IF wqi_low_share >= 0.30 FOR 2 连续周 THEN
    创建任务【水环境4周快治】，责任=水务/城管；目标：4周内 wqi_low_share < 0.20
    IF supply_block_rate > 0.70 FOR 3 天 THEN
    创建任务【绿色通道+多式联运】，责任=交通/军警；目标：14天内 < 0.60
    IF mh_reach_per_10k < 10 AND displaced > 50000 THEN
    创建任务【心理卫生军团化】，责任=卫健/教育/NGO；目标：30天内 ≥ 15
    IF co2_per_household > 150 THEN
    创建任务【低碳重建合规审计】，责任=住建/承包商；目标：90天内 ≤ 120
    ```
  - 合格阈值：至少4 条触发器，含阈值+持续期+动作+目标+时限。
- 小标准3.2.A.2【1分｜完备性】任务流与依赖
  - 用文字或流程图描述采集→计算→告警→派单→执行→复核链路，标注责任单位与SLA（如：告警触发后 2h 内派单，24h 回执，7d 复核）。
  - 合格阈值：六个环节都出现，且每个环节至少指明一个责任主体与时限。

#### 路径3.2.B【2分 | 轻量化调度脚本（伪代码）】
- 示例（Python 伪代码）：
  ```python
  run_daily('06:00', tasks=[
  compute_kpis(), # 汇总KPI
  check_thresholds(), # 比对红/黄/绿
  create_tickets_if_needed(), # 生成任务并推送至指挥系统
  update_dashboard() # 刷新可视化
  ])
  ```
- 合格阈值：包含计划时间、任务序列、失败重试/告警（例如：失败重试 3 次→短信通知值班长）。
