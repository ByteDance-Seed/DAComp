### 标准1.1（最高 6分）：时间窗口与样本口径的选择与校验
#### 路径1.1.A（推荐｜“近6个月 = 近期3个月 vs 前期3个月”差分法）
- 1.1.A.1【1分｜完备性】说明 `dt_max = 2025-01-25 13:00:49`（源自 `MAX(COALESCE(updated_at,resolved_at,created_at))`），据此定义 `prior=[2024-07-25,2024-10-25)`、`recent=[2024-10-25,2025-01-25]`，并声明 `stakeholder_id ↔ assignee_user_id / reporter_user_id` 的匹配口径。
- 1.1.A.2【4分｜精确性】评分时核查以下结果与执行一致（误差=0）：`issue_prior_created=0`、`issue_recent_created=0`、`prior_assignees=0`、`recent_assignees=0`、`prior_reporters=0`、`recent_reporters=0`，且 `jira__daily_issue_field_history` 在同窗的 `date_day` 记录为 0。需明确“窗口过滤→stakeholder×period 聚合→差分”的流程，并给出可复执行 SQL 框架：
  ```sql
  WITH params AS (
    SELECT datetime('2025-01-25 13:00:49') AS dt_max,
           datetime('2025-01-25 13:00:49','-3 months') AS recent_start,
           datetime('2025-01-25 13:00:49','-6 months') AS prior_start
  )
  SELECT ...
  FROM jira__issue_enhanced, params
  WHERE DATE(created_at) BETWEEN prior_start AND dt_max
  GROUP BY stakeholder_id, period;
  ```
- 1.1.A.3【1分｜结论性】指出“窗口样本量为 0 → 差分法无法落地，需切换快照口径”的判断，并说明其业务含义（无增量可衡量趋势）。
#### 路径1.1.B（快照差值代理法｜本库默认）
- 1.1.B.1【1分｜完备性】说明窗口缺失事实（如上全部为 0），并转向 `jira__stakeholder_engagement_insights` 快照，列明关键派生字段：`breadth_depth_delta`、`participation_density_conn`、`depth_efficiency_index`、`assign_report_ratio`、`out_in_ratio`。
- 1.1.B.2【4分｜精确性】同时核查：`cohort_size=4524`、`avg_outbound_influence=17.47±0.05`、`avg_inbound_influence=13.836±0.01`、`avg_direct_network_connections=27.268±0.01`、`avg_strategic_value_score=70.853±0.01`、`avg_breadth=23.369±0.01`、`avg_depth=22.977±0.01`，并说明 `participation_density_conn = total_comments_authored/NULLIF(direct_network_connections,0)`、`depth_efficiency_index=(engagement_depth_score+engagement_quality_score)/2` 等计算。需提供可复跑 SQL 轮廓：
  ```sql
  WITH base AS (
    SELECT *,
           engagement_breadth_score - engagement_depth_score AS breadth_depth_delta,
           CASE WHEN direct_network_connections>0 THEN total_comments_authored*1.0/direct_network_connections END AS participation_density_conn,
           (engagement_depth_score + engagement_quality_score)/2.0 AS depth_efficiency_index
    FROM jira__stakeholder_engagement_insights
  )
  SELECT ... FROM base;
  ```
- 1.1.B.3【1分｜结论性】强调快照法用于结构化预警，不可直接推断时间趋势，需在报告中注明样本静态属性。
#### 路径1.1.C（兜底｜“近12个月拆分 2×6个月”差分法）
- 1.1.C.1【1分｜完备性】若需说明兜底路径，给出 `prior=[2024-01-25,2024-07-25)`、`recent=[2024-07-25,2025-01-25]` 的定义与适用条件。
- 1.1.C.2【4分｜精确性】确认 `issues_prior=0`、`issues_recent=0`，并说明计算流程与 1.1.A 类似（窗口换成 6 个月）。
- 1.1.C.3【1分｜结论性】指出“12 个月差分亦为空 → 仍需回退快照”以保证分析连贯。
